# IMU 手套嵌入式对接文档

## 1. 系统概述

### 1.1 硬件架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        手套端 (Sender)                           │
│  ┌─────────────┐    I2C     ┌─────────────────────────────────┐ │
│  │ XIAO ESP32S3│◄──────────►│  11x WT901 IMU 传感器            │ │
│  │  (主控)      │  SDA=GPIO8 │  地址: 0x50~0x60                │ │
│  │              │  SCL=GPIO9 │                                 │ │
│  └──────┬───────┘  400kHz   └─────────────────────────────────┘ │
│         │ USB                                                    │
└─────────┼────────────────────────────────────────────────────────┘
          │
          ▼
    ┌───────────┐     USB      ┌──────────────┐
    │  电脑 PC   │◄────────────│  接收器/直连  │
    │  SDK 软件  │  /dev/ttyACM│  (ESP32小板)  │
    └───────────┘              └──────────────┘
```

### 1.2 两种工作模式

| 模式 | 连接方式 | 波特率 | 数据格式 | 用途 |
|------|----------|--------|----------|------|
| **Text 模式** | 手套 ESP32 USB 直连电脑 | 921600 | 文本行 | 调试、校准 |
| **Binary 模式** | ESP-NOW 无线 → 接收器 → USB | 2000000 | 168字节二进制帧 | 正式使用 |

## 2. IMU 传感器配置

### 2.1 I2C 参数

| 参数 | 值 |
|------|-----|
| SDA | GPIO8 |
| SCL | GPIO9 |
| 频率 | 400 kHz |
| 地址范围 | 0x50 ~ 0x60 (共 11 个) |

### 2.2 IMU 地址与关节映射

```
┌──────────┬──────────┬────────────────────────────────┐
│ I2C 地址 │ 关节编号 │ 位置                            │
├──────────┼──────────┼────────────────────────────────┤
│  0x50    │    0     │ 腕部 (Wrist)                    │
│  0x51    │   13     │ 拇指 MCP (Thumb Metacarpal)     │
│  0x52    │   14     │ 拇指 PIP (Thumb Proximal)       │
│  0x53    │    1     │ 食指 MCP (Index Metacarpal)     │
│  0x54    │    2     │ 食指 PIP (Index Proximal)       │
│  0x55    │    4     │ 中指 MCP (Middle Metacarpal)    │
│  0x56    │    5     │ 中指 PIP (Middle Proximal)      │
│  0x57    │   10     │ 小指 MCP (Little Metacarpal)    │
│  0x58    │   11     │ 小指 PIP (Little Proximal)      │
│  0x59    │    7     │ 无名指 MCP (Ring Metacarpal)    │
│  0x60    │    8     │ 无名指 PIP (Ring Proximal)      │
└──────────┴──────────┴────────────────────────────────┘
```

**注意**: DIP 关节（指尖）由 PIP 关节耦合推算，无独立 IMU。

### 2.3 WT901 寄存器

| 寄存器 | 地址 | 说明 |
|--------|------|------|
| REG_ROLL | 0x3D | 欧拉角起始 (Roll/Pitch/Yaw, 各 2 字节) |
| REG_Q0 | 0x51 | 四元数起始 (W/X/Y/Z, 各 2 字节) |
| REG_KEY | 0x69 | 解锁密钥寄存器 |
| REG_CALSW | 0x01 | 校准模式切换 |
| REG_SAVE | 0x00 | 保存校准到 Flash |

## 3. Text 模式协议 (IMU_Serial_Direct 固件)

### 3.1 串口参数

| 参数 | 值 |
|------|-----|
| 波特率 | 921600 |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验 | 无 |
| 帧率 | ~30 Hz (delay 33ms) |

### 3.2 帧格式

每帧以文本行输出，结构如下：

```
==== IMU Direct Reader
Frame: 123  Bat: 85%
[0x50] Roll:   -29.85 Pitch:   -27.34 Yaw:   133.64 Q:   0.4255   0.1114  -0.3199   0.8391
[0x51] Roll:   153.05 Pitch:     3.99 Yaw:  -111.98 Q:  -0.1021  -0.5503   0.8010   0.2122
...
[0x60] Roll:   xxx.xx Pitch:   xxx.xx Yaw:   xxx.xx Q:   x.xxxx   x.xxxx   x.xxxx   x.xxxx
```

### 3.3 字段说明

| 字段 | 说明 |
|------|------|
| `==== IMU Direct Reader` | 帧头标识 |
| `Frame: xxx` | 帧序号 |
| `Bat: xx%` | 电池百分比 |
| `[0xXX]` | IMU I2C 地址 |
| `Roll/Pitch/Yaw` | 欧拉角 (度)，int16 / 32768 × 180 |
| `Q: w x y z` | 四元数，int16 / 32768 |

### 3.4 数据转换

```c
// WT901 原始数据为 int16，转换公式：
float roll  = raw_euler[0] / 32768.0f * 180.0f;  // 度
float pitch = raw_euler[1] / 32768.0f * 180.0f;
float yaw   = raw_euler[2] / 32768.0f * 180.0f;

float qw = raw_quat[0] / 32768.0f;
float qx = raw_quat[1] / 32768.0f;
float qy = raw_quat[2] / 32768.0f;
float qz = raw_quat[3] / 32768.0f;
```

## 4. Binary 模式协议 (ESP-NOW)

### 4.1 串口参数

| 参数 | 值 |
|------|-----|
| 波特率 | 2000000 |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验 | 无 |
| 帧大小 | 固定 168 字节 |

### 4.2 帧结构 (168 字节)

```
┌─────────┬─────────┬───────────┬──────────┬───────────┬────────────────────┬──────────────┬─────────┐
│  Magic  │ hand_id │ imu_count │  bat %   │ frame_id  │   IMU Data [11]    │ wrist_gyro   │   XOR   │
│ 2 bytes │ 1 byte  │  1 byte   │ 1 byte   │ 2 bytes   │    154 bytes       │  6 bytes     │ 1 byte  │
│ 0xAA 0x55│  0/1    │    11     │  0~100   │ 小端 uint16│  每路 14 字节       │  3×int16     │ 校验和  │
└─────────┴─────────┴───────────┴──────────┴───────────┴────────────────────┴──────────────┴─────────┘
  [0:2]      [2]       [3]         [4]        [5:7]          [7:161]            [161:167]     [167]
```

### 4.3 各字段详解

#### 4.3.1 帧头 (Bytes 0-1)
```c
uint8_t magic[2] = {0xAA, 0x55};  // 固定标识
```

#### 4.3.2 手别标识 (Byte 2)
```c
uint8_t hand_id;
// 0 = 左手 (LEFT)
// 1 = 右手 (RIGHT)
```

#### 4.3.3 IMU 数量 (Byte 3)
```c
uint8_t imu_count = 11;  // 固定 11 个
```

#### 4.3.4 电池百分比 (Byte 4)
```c
uint8_t bat_percent;  // 0~100
```

#### 4.3.5 帧序号 (Bytes 5-6)
```c
uint16_t frame_id;  // 小端序 (Little-Endian)
// frame_id = bytes[5] | (bytes[6] << 8)
```

#### 4.3.6 IMU 数据 (Bytes 7-160)

11 路 IMU，每路 14 字节：

```c
typedef struct {
    int16_t euler[3];  // Roll, Pitch, Yaw (原始值)
    int16_t quat[4];   // W, X, Y, Z (原始值)
} ImuRaw;  // 14 bytes

ImuRaw imu_data[11];  // 按地址 0x50~0x60 顺序排列
```

**字节序**: 小端 (Little-Endian)

**数据转换**:
```c
// 欧拉角转浮点 (度)
float roll  = imu_data[i].euler[0] / 32768.0f * 180.0f;
float pitch = imu_data[i].euler[1] / 32768.0f * 180.0f;
float yaw   = imu_data[i].euler[2] / 32768.0f * 180.0f;

// 四元数转浮点
float qw = imu_data[i].quat[0] / 32768.0f;
float qx = imu_data[i].quat[1] / 32768.0f;
float qy = imu_data[i].quat[2] / 32768.0f;
float qz = imu_data[i].quat[3] / 32768.0f;

// 归一化
float norm = sqrtf(qw*qw + qx*qx + qy*qy + qz*qz);
if (norm > 1e-6f) {
    qw /= norm; qx /= norm; qy /= norm; qz /= norm;
}
```

#### 4.3.7 腕部陀螺仪 (Bytes 161-166)
```c
int16_t wrist_gyro[3];  // X, Y, Z 角速度 (原始值)
```

#### 4.3.8 XOR 校验 (Byte 167)
```c
uint8_t xor_checksum = 0;
for (int i = 0; i < 167; i++) {
    xor_checksum ^= frame[i];
}
// 校验: xor_checksum == frame[167]
```

### 4.4 C 语言帧结构定义

```c
#define BINARY_MAGIC_0      0xAA
#define BINARY_MAGIC_1      0x55
#define BINARY_FRAME_SIZE   168
#define BINARY_IMU_SIZE     14
#define IMU_COUNT           11

typedef struct __attribute__((packed)) {
    // 帧头
    uint8_t  magic[2];        // [0:2]   0xAA, 0x55
    
    // 基本信息
    uint8_t  hand_id;         // [2]     0=左, 1=右
    uint8_t  imu_count;       // [3]     11
    uint8_t  bat_percent;     // [4]     0~100
    uint16_t frame_id;        // [5:7]   小端
    
    // IMU 数据
    struct __attribute__((packed)) {
        int16_t euler[3];     // Roll, Pitch, Yaw
        int16_t quat[4];      // W, X, Y, Z
    } imu[IMU_COUNT];         // [7:161] 11 × 14 = 154 字节
    
    // 腕部陀螺仪
    int16_t  wrist_gyro[3];   // [161:167]
    
    // 校验
    uint8_t  xor_checksum;    // [167]
} SerialFrame;
```

## 6. 磁力计校准流程

### 6.1 新版流程（推荐）

当前产品固件（30Hz Product）已在左右手主程序中集成 USB 触发磁力计校准，**不需要切换固件**。

校准命令通过手套 USB 串口发送（波特率 921600）：

```
$GLOVITY,CAL,START    # 开始磁力计校准
$GLOVITY,CAL,STOP     # 停止校准并保存
$GLOVITY,CAL,STATUS   # 查询校准状态
$GLOVITY,NORMAL       # 回到正常工作模式
```

校准步骤：

1. 手套保持 30Hz Product 固件，通过 USB 连接电脑
2. 电脑端发送 `$GLOVITY,CAL,START` 命令
3. 用户拿着手套在空中缓慢做 8 字形旋转，约 30~35 秒
4. 电脑端发送 `$GLOVITY,CAL,STOP` 命令
5. 固件自动保存校准数据到 IMU Flash，并回到 NORMAL 模式

串口交互示例：

```
发送: $GLOVITY,CAL,START
收到: $GLOVITY,CAL,STARTED
... (用户旋转约 35 秒) ...
发送: $GLOVITY,CAL,STOP
收到: $GLOVITY,CAL,COMPLETE
收到: $GLOVITY,MODE,NORMAL
```

## 7. 电池电压读取

```c
#define BAT_ADC_PIN   2      // GPIO2
#define R_UPPER       10000.0f
#define R_LOWER       22000.0f
#define BAT_FULL_MV   4200
#define BAT_EMPTY_MV  3000

uint8_t readBatPercent() {
    uint32_t adc_mv = analogReadMilliVolts(BAT_ADC_PIN);
    uint32_t bat_mv = (uint32_t)(adc_mv * (R_UPPER + R_LOWER) / R_LOWER);
    if (bat_mv >= BAT_FULL_MV)  return 100;
    if (bat_mv <= BAT_EMPTY_MV) return 0;
    return (uint8_t)((bat_mv - BAT_EMPTY_MV) * 100 / (BAT_FULL_MV - BAT_EMPTY_MV));
}
```

## 8. 常见问题

### 8.1 I2C 扫描不到 IMU

- 检查手套电池是否开启（IMU 需要电池供电）
- 检查 I2C 接线：SDA=GPIO8, SCL=GPIO9
- 确认 ESP32 与 IMU 共地

### 8.2 数据全为 0

- IMU 未初始化完成，上电后需等待 ~1 秒
- 检查 I2C 地址是否正确 (0x50~0x60)

### 8.3 四元数不归一

- WT901 原始 int16 值需除以 32768 转浮点
- 接收端需做归一化处理

### 9.3 Python 解析示例

```python
import serial
import struct

# Binary 模式解析
def parse_binary_frame(frame: bytes) -> dict:
    """解析 168 字节二进制帧"""
    if len(frame) != 168:
        return None
    
    # XOR 校验
    xor = 0
    for b in frame[:-1]:
        xor ^= b
    if xor != frame[-1]:
        return None
    
    # 解析字段
    hand_id = frame[2]
    imu_count = frame[3]
    bat = frame[4]
    frame_id = frame[5] | (frame[6] << 8)
    
    # 解析 IMU 数据
    imu_data = []
    for i in range(11):
        off = 7 + i * 14
        vals = struct.unpack_from('<7h', frame, off)
        euler = vals[:3]
        quat = vals[3:]
        
        # 转浮点
        qw = quat[0] / 32768.0
        qx = quat[1] / 32768.0
        qy = quat[2] / 32768.0
        qz = quat[3] / 32768.0
        
        imu_data.append({
            'addr': 0x50 + i,
            'euler': euler,
            'quat': (qw, qx, qy, qz)
        })
    
    return {
        'hand_id': hand_id,
        'bat': bat,
        'frame_id': frame_id,
        'imu': imu_data
    }
```
